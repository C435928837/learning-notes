## 给你两个按 非递减顺序 排列的整数数组 nums1 和 nums2，另有两个整数 m 和 n ，分别表示 nums1 和 nums2 中的元素数目。请你 合并 nums2 到 nums1 中，使合并后的数组同样按 非递减顺序 排列。

核心思路是：从后往前比较，把较大的值放到 nums1 的最后。如果从前往后放，会覆盖 nums1 中还没比较的元素。

public void merge(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1;
    int j = n - 1;
    int k = m + n - 1;

    while (j >= 0) {
        if (i >= 0 && nums1[i] > nums2[j]) {
            nums1[k] = nums1[i];
            i--;
        } else {
            nums1[k] = nums2[j];
            j--;
        }
        k--;
    }
}

为什么循环条件只写 j >= 0？
如果 nums2 已经全部放完：
nums1 剩余元素本来就在正确位置，不需要移动。

如果 nums1 的有效元素先用完：
继续把 nums2 剩余元素复制进 nums1。
