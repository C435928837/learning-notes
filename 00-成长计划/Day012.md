## 3. CAS

CAS 全称：

```text
Compare And Set
```

核心代码：

```java
stock.compareAndSet(expectedValue, newValue);
```

三个值：

| 名称 | 含义         |
| --- |------------|
| 当前值 | 变量的当前值     |
| 预期值 | 符合预期值才做修改  |
| 新值 | 需要把变量修改成的值 |

CAS 执行过程：

```text
当前值 == 预期值：
    更新为新值，返回 true

当前值 != 预期值：
    不更新，返回 false
```

## 4. CAS 自旋

CAS 失败的原因：

>其他线程已经修改了变量的值，导致变量的当前值与预期值不一致

CAS 自旋示例：

```java
while (true) {
    int currentStock = stock.get();

    if (currentStock <= 0) {
        return false;
    }

    if (stock.compareAndSet(currentStock, currentStock - 1)) {
        return true;
    }
}
```

谁负责重试：

| 方法 | 是否需要业务代码手写重试 |
| --- |--------------|
| `incrementAndGet()` | 不需要          |
| `decrementAndGet()` | 不需要          |
| `compareAndSet()` | 需要           |
| “库存大于 0 才扣减” | 需要           |

CAS 的优点：

-无需显式加锁
-低竞争下性能较好；适合单变量原子更新

CAS 的问题：

- 高并发下可能反复失败：

## 5. AtomicInteger 常用方法

```java
AtomicInteger count = new AtomicInteger(10);
```

| 方法 | 作用                                         | 返回值     |
| --- |--------------------------------------------|---------|
| `get()` | 获取当前值                                      | int     |
| `set(value)` | 设置当前值                                      | void    |
| `incrementAndGet()` | 先自增1再获取                                    | int     |
| `getAndIncrement()` | 先获取再自增1                                    | int     |
| `decrementAndGet()` | 先自减1再获取                                    | int     |
| `getAndDecrement()` | 先获取再自减1                                    | int     |
| `addAndGet(delta)` | 先自增指定值再获取                                  | int     |
| `getAndAdd(delta)` | 先获取再自增指定值                                  | int     |
| `compareAndSet(expected, update)` | 判断当前值是否等于预期值，等于把当前值修改成新值并返回true，不等于返回false | boolean |

`incrementAndGet()` 与 `getAndIncrement()` 的区别：

>先自增1再获取和先获取再自增1

## 6. volatile、AtomicInteger、synchronized 对比

| 方案 | 可见性 | 原子性 | 适用场景                       |
| --- |-----|-----|----------------------------|
| `volatile` | 有   | 有   | 只需要可见性                     |
| `AtomicInteger` | 无   | 有   | 单个int变量做原子操作               |
| `synchronized` | 有   | 有   | 多个变量做原子操作或需要临界区保证线程安全做复杂业务 |

我的选型结论：

>只需要可见性选择volatile，单个变量做原子操作选择AtomicInteger，多个变量做原子操作或需要临界区保证线程安全做复杂业务选择synchronized

## 9. 算法题：合并两个有序数组

题目：

```text
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6], n = 3

结果：[1,2,2,3,5,6]
```

解题思路：

```text
使用三个指针：
i：nums1 有效元素末尾
j：nums2 元素末尾
k：nums1 总数组末尾
```

为什么从后往前合并：

>如果从前往后放，会覆盖nums1中还没比较的元素。

时间复杂度：`O(M+N)`  
空间复杂度：`O(1)`

代码：

```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        int i = m - 1;
        int j = n - 1;
        int k = m + n - 1;

        while (j >= 0) {
            if (i >= 0 && nums1[i] > nums2[j]) {
                nums1[k] = nums1[i];
                i--;
            } else {
                nums1[k] = nums2[j];
                j--;
            }
            k--;
        }
    }
}
```

## 10. 高频面试题

### 10.1 AtomicInteger 是什么？为什么可以保证原子更新？

>AtomicInteger通过volatile保证可见性，并借助CAS保证单个变量更新的原子性

### 10.2 CAS 的执行过程是什么？

>判断当前值是否等于预期值，等于把当前值修改成新值并返回true，不等于返回false

### 10.3 为什么 `if (stock.get() > 0) { stock.decrementAndGet(); }` 仍可能超卖？

>因为判断和赋值这两个操作是独立原子操作，当存在两个线程执行时，可能两个线程都同时通过了判断条件进去赋值操作

### 10.4 volatile、AtomicInteger、synchronized 如何选择？

>只需要可见性选择volatile，单个变量做原子操作选择AtomicInteger，多个变量做原子操作或需要临界区保证线程安全做复杂业务选择synchronized
