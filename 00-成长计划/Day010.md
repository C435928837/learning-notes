## 3. 共享变量与库存超卖

共享变量示例：

```java
private int stock = 1;
```

不安全的扣减逻辑：

```java
if (stock > 0) {
    stock--;
    return true;
}
```

`stock--` 实际包含：

```text
1. 读取 stock
2. 计算 stock - 1
3. 写回 stock
```

超卖发生过程：

```text
线程 A 读取 stock = 1
线程 B 读取 stock = 1
线程 A 扣减后写回 0，下单成功
线程 B 基于旧值扣减后写回 0，下单成功
最终：库存为 0，但成功订单有 2 个
```

我的理解：

>多线程可以同时读取共享变量，也可以按顺序修改共享变量，但如果同时修改共享变量，就有可能会产生线程安全问题。

## 4. synchronized

### 4.1 synchronized 是什么

`synchronized` 的作用：

- 是java的关键字，synchronized以某个对象作为锁对象，JVM使用该对象关联的Monitor实现互斥
- 保证原子性、可见性、有序性
- 可以锁对象、实例、类

它锁住的本质是：

```text
某个对象关联的 Monitor
```

### 4.2 同一把锁的互斥现象

```java
private final Object lock = new Object();

public void handle() {
    synchronized (lock) {
        // 临界区
    }
}
```

当线程 A 已持有 `lock` 时，线程 B 再尝试进入同一个 `synchronized(lock)` 代码块会：

```text
线程 B 状态：等待锁释放
线程 A 释放锁后：与其他线程竞争锁
```

### 4.3 synchronized 的三种常见写法

同步代码块：

```java
synchronized (lock) {
    // 临界区
}
```

同步实例方法：

```java
public synchronized void method() {
}
```

等价写法：

```java
public void method() {
    synchronized (this) {
    }
}
```

同步静态方法：

```java
public static synchronized void method() {
}
```

等价写法：

```java
synchronized (类名.class) {
}
```

## 5. 对象锁、实例锁、类锁

| 类型 | 锁对象     | 写法                             | 适合保护的数据 |
| --- |---------|--------------------------------|-------|
| 实例锁 | 实例      | synchronized(this)或者同步实例方法     | 实例变量  |
| 对象锁 | 对象      | synchronized(object) | 共享变量  |
| 类锁 | class对象 | synchronized(类名.class)或者同步静态方法   |   静态共享变量    |

需要记住：

```text
同一个锁对象：互斥
不同锁对象：不互斥
```

以下代码中的锁是否相同，为什么？

```java
synchronized (this) {
    stock--;
}

synchronized (stockLock) {
    stock--;
}
```

答案：

>不相同，因为一个锁的是实例，一个锁的是stockLock对象

## 6. 可重入性

示例：

```java
public synchronized void methodA() {
    methodB();
}

public synchronized void methodB() {
}
```

为什么线程不会把自己阻塞？

>当线程获得对象的锁后，就不用再重新竞争锁。

我的理解：

> `synchronized` 是对象锁。同一个线程已经拿到某个对象的锁后，就不用再重新竞争锁。

## 7. JMM

JMM 全称：

```text
Java Memory Model
```

它解决的问题：

>多线程内存管理规范

| 特性 | 含义                 | 典型问题 | 常见解决方式        |
| --- |--------------------|------|---------------|
| 可见性 | 其他线程可以看到某个线程修改后的结果 | 线程停止标记改了，另一个线程仍看不到 | synchronized、volatile |
| 原子性 | 运行期间不会被其他线程插入      | stock--、i++ 导致丢失更新和超卖     |     synchronized          |
| 有序性 | 防止出现不符合预期的指令重排     |  双重检查锁单例可能拿到未初始化完成的对象 |  synchronized、volatile             |

重点结论：

```text
volatile：保证可见性和有序性，不保证原子性。
synchronized：保证可见性、原子性和有序性。
```

## 9. 高频面试题

### 9.1 synchronized 是什么？为什么能实现锁？

>是java的关键字，锁的不是对象，而是对象的Monitor

### 9.2 synchronized 锁的到底是什么？

>可以锁对象、实例和类

### 9.3 synchronized(this)、synchronized(lock)、synchronized(类名.class) 有什么区别？

>synchronized(this)锁的是实例，保护对象变量，synchronized(lock)锁的是对象，保护共享变量，synchronized(类名.class)，保护静态共享变量。

### 9.4 为什么 volatile 不能解决库存扣减超卖？

>不能保证原子性

### 9.5 synchronized 的可重入性是什么？

>同一线程已持有某个对象的 Monitor 后，可以再次进入该对象的同步代码；JVM 会记录重入次数，退出相同次数后才真正释放锁。

### 9.6 synchronized 能解决分布式多实例部署下的超卖吗？为什么？

>不能，synchronized只对同一个JVM的对象有效
