## 2. 线程生命周期

| 状态 | 含义 | 常见触发方式 |
| --- | --- | --- |
| `NEW` | 已创建，未启动 | new Thread() |
| `RUNNABLE` | 可运行或正在运行 | 调用 start() 后 |
| `BLOCKED` | 等待获取 synchronized 的 Monitor 锁 | 其他线程持有同一把 synchronized 锁 |
| `WAITING` | 无限期等待其他线程唤醒 | wait()、join()、LockSupport.park() |
| `TIMED_WAITING` | 在指定时间内等待 | sleep()、wait(timeout)、join(timeout) |
| `TERMINATED` | 线程执行结束 | run() 正常结束或抛出未捕获异常 |

重点：

```text
Java 没有单独的 RUNNING 状态；
正在运行和等待 CPU 调度的线程，都属于 RUNNABLE。
```

`BLOCKED`、`WAITING`、`TIMED_WAITING` 的区别：

>BLOCKED：等待进入 synchronized 临界区，等待的是 Monitor 锁。WAITING：已经主动放弃执行，等待其他线程执行 notify、结束或解除 park。 TIMED_WAITING：等待一段指定时间，时间到后可以自动恢复。

## 3. start() 与 run()

```java
Thread thread = new Thread(() -> {
    System.out.println(Thread.currentThread().getName());
});
```

```java
thread.start();
```

```java
thread.run();
```

两者区别：

>start()新线程执行，run()main 线程普通方法调用

线程是否可以重复调用 `start()`：

>不能，只能调用一次，再次调用会抛出异常

## 4. 三种线程任务创建方式

| 方式 | 返回值 | 受检异常 | 适用场景 |
| --- | --- | --- | --- |
| `Thread` | 无 | 不直接支持 | 学习生命周期即可，少直接使用 |
| `Runnable` | 无 | 不直接支持 | 普通异步任务 |
| `Callable<V>` | 有 | 可以抛出 | 需要任务结果或异常处理 |

### 4.1 Thread

```java
class MyThread extends Thread {
    @Override
    public void run() {
        // 自己实现
    }
}
```

缺点：

>是Java单继承。继承了Thread后，类不能再继承其他业务父类

### 4.2 Runnable

```java
Runnable task = () -> {
    // 普通异步任务
};

Thread thread = new Thread(task);
thread.start();
```

适合场景：

>异步记录操作日志、发送普通通知

### 4.3 Callable

```java
Callable<Integer> task = () -> {
    // 可以返回结果，也可以抛出异常
    return 0;
};
```

适合场景：

>异步查询商品、库存、优惠券信息，最后获取聚合结果

## 5. Future

`Future` 的作用：

>异步任务的结果凭证

常用方法：

| 方法 | 作用                                        |
| --- |-------------------------------------------|
| `get()` | 获取结果，未完成则一直等待                             |
| `get(timeout, unit)` | 在指定时间内等待结果                                |
| `isDone()` | 任务是否已经结束，不区分成功、失败、取消                      |
| `isCancelled()` | 任务是否已被取消                                  |
| `cancel(boolean)` | cancel(false)：尝试取消，但不会中断正在运行的线程。cancel(true)：尝试取消；若任务正在运行，会发送中断信号。 |

`Future.get()` 为什么会阻塞：

>调用 get() 的线程会等待

任务执行异常时：

>抛出 ExecutionException

## 6. Thread、Runnable、Callable、FutureTask、Future 的关系

```text
Runnable / Callable：任务
Thread：执行任务的线程
Future：异步任务结果的凭证
FutureTask：既能执行任务，也能保存结果的 Future 实现
```

我的理解：

>实际项目中通常将 Runnable 或 Callable 提交给线程池，而不是频繁直接 new Thread()

## 9. 高频面试题

### 9.1 `start()` 和直接调用 `run()` 有什么区别？

>start() 会创建一条新的执行线程，并由 JVM 在新线程中调用 run()；直接调用 run() 只是普通方法调用，仍然由当前线程执行，不会产生并发。一个 Thread 对象只能调用一次 start()，重复调用会抛出 IllegalThreadStateException。

### 9.2 Thread、Runnable、Callable 有什么区别？

>实际项目中通常将 Runnable 或 Callable 提交给线程池，而不是频繁直接 new Thread()。
> 
| 方式 | 返回值 | 受检异常 | 使用场景 |
| --- | --- | --- | --- |
| `Thread` | 无 | 不直接支持 | 学习线程基础，项目中很少直接继承 |
| `Runnable` | 无 | 不直接支持 | 普通异步任务 |
| `Callable<V>` | 有 | 可以抛出 | 需要异步计算结果或异常处理 |

### 9.3 为什么不建议频繁 `new Thread()`？

>线程创建和销毁需要消耗系统资源；大量线程会占用内存、增加上下文切换成本，也无法统一控制线程数量、队列、异常处理和拒绝策略。线程池可以复用线程，并限制并发量，避免系统资源被耗尽。

### 9.4 Future 是什么？`Future.get()` 为什么会阻塞？

>Future 是异步任务结果的凭证。任务提交后可以先继续执行其他逻辑，之后通过 Future 查询状态、获取结果或取消任务。若任务尚未结束，调用 get() 的线程会等待，因此可能阻塞；任务异常时，get() 会抛出 ExecutionException。

### 9.5 Runnable 提交到线程池后，`Future.get()` 返回什么？

>Runnable 本身没有返回值，所以 future.get() 正常完成后返回 null。只有提交 Callable 或带返回值的任务时，Future.get() 才能获得业务结果。

### 9.6 Future 和 FutureTask 有什么区别？

>Future 是接口，定义了获取结果、取消任务和查询状态等能力；FutureTask 是它的实现类，既可以作为 Runnable 被线程或线程池执行，也可以保存异步任务结果。项目中更常见的是通过线程池 submit() 直接获得 Future。

### 9.7 BLOCKED、WAITING、TIMED_WAITING 有什么区别？

>BLOCKED 是等待获取 synchronized 的 Monitor 锁；WAITING 是无限期等待其他线程唤醒或结束，例如 wait()、join()；TIMED_WAITING 是带超时时间的等待，例如 sleep()、wait(timeout)、join(timeout)。

## 10. 算法题：二分查找

题目：

```text
给定一个升序数组 nums 和目标值 target，
若 target 存在则返回下标，否则返回 -1。
```

示例：

```text
nums = [-1, 0, 3, 5, 9, 12]
target = 9
结果：4
```

解题思路：

```text
使用左右双指针：
left 指向查找范围左边界；
right 指向查找范围右边界；
每次比较中间值与 target，缩小一半范围。
```

循环不变量：

>每次循环开始时，若 target 存在，则它一定在 [left, right] 的闭区间内。

时间复杂度：`O(logn)`  
空间复杂度：`O(1)`

代码：

```java
class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        while(left <= right){
            int mid = (left + right) / 2;
            if(target == nums[mid]){
                return mid;
            } else if(nums[mid] > target){
                right = mid - 1;
            } else if(nums[mid] < target){
                left = mid + 1;
            }
        }
        return -1;
    }
}
```
