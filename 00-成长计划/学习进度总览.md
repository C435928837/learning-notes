## 3. 已掌握的面试表达

### Java 与集合

- 四大特性：封装、继承、多态、抽象。
- == 与 equals()：引用比较与业务相等性；重写 equals() 时通常也要重写 hashCode()。
- String 不可变：final 类、内部数据不暴露、无修改 API，支持安全共享与稳定哈希。
- ArrayList：底层动态数组，查询 O(1)，中间增删需要移动元素；正常尾插均摊 O(1)。
- HashSet：底层基于 HashMap，元素作为 Key，通过 hashCode() 定位、equals() 确认是否重复。

### 并发基础

- 库存超卖：判断库存和扣减库存不是原子操作，多个线程交错执行会造成重复下单。
- synchronized：锁住对象关联的 Monitor；同一锁对象互斥；可保证原子性、可见性和有序性；只在单 JVM 内有效。
- volatile：保证可见性和有序性，不保证 stock--、i++ 等复合操作的原子性。
- AtomicInteger：通过 volatile 维护可见性，并使用 CAS 完成单变量原子更新；复杂业务不能只依赖单个原子变量。
- 双重检查锁单例：第一层 if 减少锁竞争，第二层 if 避免重复创建，synchronized 保证互斥，volatile 避免半初始化对象提前发布。
- Future：异步结果凭证；get() 在任务未完成时会阻塞，异常任务通过 ExecutionException 暴露。

### 笔记复查

- Day012 的对比表中，volatile 的“原子性”应为“无”；其正文结论“volatile 不保证原子性”是正确的。
- CAS 适合单变量原子更新；“判断库存大于 0 再扣减”必须使用 CAS 循环，不能拆成 get() 判断加 decrementAndGet()。
- LinkedList 的中间增删只有在已定位节点时才是 O(1)；按下标定位通常仍是 O(n)。
