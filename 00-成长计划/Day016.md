## 2. ThreadLocal 是什么

我的理解：

>作用于线程的变量，不保证线程的安全，确保每个线程的变量是独立的

核心结论：

>`ThreadLocal` 不是用于给共享变量加锁，而是 `给每个线程的变量单独赋值`。

适用场景：

- 当前用户 ID：
- 租户 ID：
- TraceId：
- 其他：线程的调用链路

不适用场景：

>共享公共变量

## 3. 数据副本存在哪里

简化结构：

```text
ThreadLocal 对象
     |
线程 A 的 ThreadLocalMap -> key: ThreadLocal, value: 线程A的内容空间
线程 B 的 ThreadLocalMap -> key: ThreadLocal, value: 线程B的内存空间
```

我的理解：

>`ThreadLocal` 本身 `是一个引用对象`；数据实际保存在 `ThreadLocalMap的value`。

为什么不同线程可以通过同一个 `ThreadLocal` 对象读取不同值：

>实际上ThreadLocal通过引用对象的弱引用作为key，把值存到对应的value

## 4. set、get、remove

### 4.1 set()

```java
USER_ID.set(userId);
```

作用：

>设置当前线程的threadlocal变量的值为userId

### 4.2 get()

```java
String userId = USER_ID.get();
```

作用：

>获取当前线程的threadlocal变量的值

未设置值时：

>返回null

### 4.3 remove()

```java
USER_ID.remove();
```

作用：

>移除当前线程中、当前 ThreadLocal 对应的 Entry；执行过程中可能顺带清理失效 Entry，但不是它的主要语义

推荐写法：

```java
USER_ID.set(userId);
try {
    // 处理当前任务
} finally {
    USER_ID.remove();
}
```

为什么必须在 `finally` 中调用 `remove()`：

>移除当前线程中、当前 ThreadLocal 对应的 Entry；执行过程中可能顺带清理失效 Entry，但不是它的主要语义

## 5. ThreadLocal 内存泄漏与上下文污染

`ThreadLocalMap` 的 Entry：

```text
Entry
├── key：ThreadLocal 的弱引用
└── value：业务数据的强引用
```

Key 使用弱引用的原因：

>避免线程长期活动时，已经不用的ThreadLocal对象被Map一直强引用无法回收

Key 被 GC 回收后，Entry 的状态：

```text
key：null
value：值
```

为什么 Value 仍可能无法及时回收：

>value仍被map引用，不能被cg回收

线程池中的上下文污染：

>ThreadLocal 被静态变量强引用，Key 根本不会被 GC 回收；如果任务结束后不 remove()，同一线程后续任务仍能 get() 到旧值

结论：

>`在线程池中，ThreadLocal 使用完毕后必须在 finally 中调用 remove，因为remove可以清除当前引用的值和引用实效的值。`

## 7. 算法题

题目：无重复字符的最长子串

题型：给定一个字符串 s ，请你找出其中不含有重复字符的 最长 子串 的长度。

核心思路：

>right 从左到右遍历字符串。 lastIndexMap 记录每个字符最后一次出现的下标。 如果当前字符以前出现过，并且它还在当前窗口内，就把 left 移到它上一次出现位置的后一位。 每轮计算当前窗口长度：right - left + 1

关键边界：

- 空字符串：
- 重复字符：
- 左边界移动：
- 最大长度更新：

时间复杂度：`O(n)`  
空间复杂度：`O(n)`

## 8. 高频面试题

### 8.1 ThreadLocal 是什么？解决了什么问题？

>ThreadLocal作用于线程的变量，不保证线程的安全，确保每个线程的变量是独立的

### 8.2 ThreadLocal 的数据存在哪里？

>每个线程的内存空间

### 8.3 为什么 ThreadLocalMap 的 Key 使用弱引用？

>避免threadlocal在不用时因线程长期工作一直被引用导致无法回收

### 8.4 为什么在线程池中必须调用 remove()？

>及时清除变量的值，避免内存泄漏和上下文污染

### 8.5 在提交线程中 set ThreadLocal，线程池工作线程能直接 get 到吗？

>不能，每个线程都是独立的threadlocalmap
