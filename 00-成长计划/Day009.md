## 2. ArrayList 核心结构

ArrayList 的本质：一个Object[]

核心字段：

```java
Object[] elementData;
int size;
```

`elementData` 的作用：ArrayList存放元素的数组

`size` 的作用：元素的长度

`size` 与 `elementData.length` 的区别：

| 名称 | 含义 |
| --- | --- |
| `size` | 元素的长度 |
| `elementData.length` | 数组的长度 |

为什么 ArrayList 可以通过下标快速查询：

因为ArrayList是基于数组实现的，本质上是数组

时间复杂度：O(1)

## 3. ArrayList 的添加与扩容

### 3.1 尾部添加

```java
list.add(element);
```

核心过程：

1. 判断元素的个数是否超过数组的长度
2. 不超长度直接添加
3. 超过长度先做扩容再做添加

尾部添加的均摊时间复杂度：O(1)

### 3.2 扩容机制

正常逐个添加元素时，首选增长量：

```java
oldCapacity >> 1
```

含义：数组长度除2

扩容后的容量通常是：数组长度的1.5倍

但扩容不一定严格是 1.5 倍，原因是：

如果指定了长度，优先按照指定长度扩容

空 ArrayList 第一次添加元素时：

默认扩容长度10

扩容为什么是 `O(n)`：

扩容是把旧数组复制到新数组上

如何减少扩容次数：预估容量并使用 new ArrayList<>(100) 或 ensureCapacity()

```java
List<Product> products = new ArrayList<>(100);
```

## 4. 指定下标插入与删除

### 4.1 指定下标插入

```java
list.add(index, element);
```

插入前：

```text
[A,B,NULL,NULL]
```

插入后：

```text
[A,X,B,NULL]
```

后续元素为什么需要右移：

因为原来的下标需要插入新元素

时间复杂度：O(n)

### 4.2 删除元素

```java
list.remove(index);
```

删除后元素为什么需要左移：

因为原来下标的元素被删除了

ArrayList 删除元素后为什么要把最后一个位置设为 `null`：

为了断开对被删除对象的引用，便于 GC 回收

时间复杂度：O(n)

## 5. ArrayList 与 LinkedList

| 对比项 | ArrayList | LinkedList |
| --- |-----------|-----|
| 底层结构 | 数组        | 链表  |
| 按下标查询 | O(1)      | O(n) |
| 尾部添加 | O(1)      | O(1) |
| 中间插入/删除 | O(n)      | 已经定位节点时O(1)，否则按下标找到找到中间节点本身仍要O(0)    |
| 内存特点 | 占用空间小     | 占用空间大 |
| 常用场景 | 需要频繁查询    | 需要频繁中间增删 |

我的选型结论：

通常业务选择ArrayList，只有已经定位节点，且频繁中间增删时，才考虑选择链表

## 7. 算法题：移动零

题目：将数组中的所有 `0` 移动到末尾，同时保持非零元素相对顺序。

示例：

```text
[0, 1, 0, 3, 12] -> [1, 3, 12, 0, 0]
```

### 7.1 冒泡排序思路

思路：通过双层遍历，每次把0已到数组最后

时间复杂度：O(n2) 
空间复杂度：O(1)

### 7.2 双指针优化

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                int temp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = temp;
                slow++;
            }
        }
    }
}
```

`fast` 指针的作用：用于遍历数组

`slow` 指针的作用：用于记录不为0的元素下标，当遇到不为0的元素就移到slow下标的位置

为什么能保持非零元素原有顺序：

fast 从左到右依次读取非零元素，slow 也按同样顺序依次放入，因此非零元素的相对顺序不变

时间复杂度：O(n)  
空间复杂度：O(1)

## 8. 面试题整理

### 8.1 ArrayList 的底层结构是什么？

答：Object[]

### 8.2 ArrayList 为什么查询快？

答：通过数组的下标查找元素

### 8.3 ArrayList 如何扩容？

答：创建ArrayList对象时，数组是一个空数组，当存放第一个元素时，默认扩容10，如果指定长度扩容，则优先按照指定长度扩容，否则按原数组长度的1.5倍扩容

### 8.4 ArrayList 中间插入或删除为什么慢？

答：因为需要复制数组，移动其他元素

### 8.5 ArrayList 和 LinkedList 怎么选？

答：通常业务选择ArrayList，只有已经定位节点，且频繁中间增删时，才考虑选择LinkedList
