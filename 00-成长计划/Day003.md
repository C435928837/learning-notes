## 1. Java 集合体系理解

### Iterable
Iterable 是什么？
是 Java 里“可以被遍历”的顶层接口。只要一个对象实现了 Iterable，就表示它可以用 for-each 遍历。
为什么实现 Iterable 的对象可以被 for-each 遍历？
Iterable的iterator()方法提供一个Iterator，Iterator里有hasNext()和next()，根据这两个方法实现遍历。
Iterable 和 Iterator 的区别是什么？
Iterable：表示“我可以被遍历”
Iterator：表示“我正在遍历”

### Collection
Collection 是什么？
Collection 是 Java 集合体系的根接口之一。是一组元素。
Collection 和 Iterable 的关系是什么？
Collection 继承 Iterable
Collection 主要提供哪些通用能力？
遍历、查询、新增、删除、判断是否存在

### List
List 的特点是什么？
有序、有下标、可重复
ArrayList 和 LinkedList 的区别是什么？
ArrayList基于数组，可快速查询，中间插入数据慢
LinkedList基于链表，查询慢，中间插入数据快
List 适合什么业务场景？
对数据要求有序可重复的场景。

### Set
Set 的特点是什么？
不可重复
HashSet、LinkedHashSet、TreeSet的区别是什么？
HashSet：基于HashMap实现，不保证顺序
LinkedHashSet：保证插入顺序
TreeSet：底层又红黑树实现，可排序
Set 适合什么业务场景？
对数据要求不可重复的场景。

### Map
Map 为什么不属于 Collection？
因为map的数据结构是key,value的形式，与collection的数据形式不一样。
Map 的 key 和 value 有什么特点？
HashMap 允许一个 null key，Hashtable 和 ConcurrentHashMap 不允许 null key / null value。
HashMap、LinkedHashMap、TreeMap、Hashtable、ConcurrentHashMap 的区别是什么？
Hashtable：线程安全，处理数据效率慢
HashMap：线程不安全，处理数据效率高
ConcurrentHashMap：线程安全
LinkedHashMap：是 HashMap 的子类，底层在 HashMap 的基础上维护了一条双向链表，因此可以保证插入顺序或访问顺序，查询效率接近 HashMap
TreeMap：底层是红黑树，会按照 key 的自然顺序或指定 Comparator 排序，插入、删除、查询复杂度是 O(log n)，适合需要按 key 排序、范围查询、获取最大最小 key 的场景。


## 2. HashMap 源码修正理解
## resize
1. resize 什么时候触发？
第一次 put 时初始化 table；后续 put 成功后，如果 size > threshold，就触发扩容。
2. capacity 和 threshold 分别是什么？
capacity代表数组的长度，threshold代表扩容的阈值。
3. 为什么第 13 次 put 会触发扩容？
因为HashMap的初始化长度是16，这时的扩容阈值是12,13大于12所以触发扩容。
4. 扩容后节点为什么只有两个位置：原位置或 j + oldCap？
HashMap 的容量始终是 2 的幂，扩容时容量会变成原来的 2 倍。下标计算公式是 hash & (capacity - 1)。扩容后 capacity - 1 的二进制只比原来多了一个高位，因此节点的新位置只取决于 hash 在 oldCap 这一位上是 0 还是 1。如果 (hash & oldCap) == 0，节点仍在原下标；否则移动到原下标 + oldCap 的位置。

##treeifyBin
1. treeifyBin 什么时候触发？
链表长度达到 8 时会触发 treeifyBin，但真正树化还要满足 table.length >= 64。
2. 链表长度 >= 8 是否一定转红黑树？
不一定，链表长度 >= 8且table.length >= 64才转红黑树
3. table.length < 64 时为什么优先扩容？
table 还小，冲突可能是桶太少导致的，扩容后节点会重新分布，可能减少链表长度；红黑树维护成本更高，所以优先扩容。
4. table.length >= 64 时为什么才树化？
table.length >= 64红黑树处理效率更优。

##afterNodeInsertion
1. HashMap 中 afterNodeInsertion 是什么作用？
空作用
2. LinkedHashMap 为什么会重写它？
LinkedHashMap 可以在插入节点后，根据 removeEldestEntry 判断是否删除最老的节点，常用于实现 LRU 缓存。
3. 它是否负责保证 key 唯一？
不是这个方法保证，是putValue()方法保证。
