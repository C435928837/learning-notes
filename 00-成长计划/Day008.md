## 3. List、Set、Queue、Map

### 3.1 List

特点：可重复、有下标

常用实现：`ArrayList`、`LinkedList`

CloudMall 场景：商品列表

### 3.2 Set

特点：不可重复

常用实现：

| 类型 | 顺序特点  | 适用场景     |
| --- |-------|----------|
| HashSet | 无序    | 无排序要求    |
| LinkedHashSet | 按插入顺序 | 需要保证插入顺序 |
| TreeSet | 按元素顺序 | 需要排序     |

CloudMall 场景：商品编码去重

### 3.3 Queue

特点：任务队列，先进先出

常用方法：

```java
queue.offer(element);
queue.poll();
queue.peek();
```

CloudMall 场景：消息任务

为什么 `ArrayDeque` 常比 `LinkedList` 更适合作栈或队列：

虽然ArrayDeque和LinkedList都能实现队列和栈，但在维护上ArrayDeque只需要维护头尾下标，而LinkedList除了要维护当前节点还要维护前后节点，维护成本高。

### 3.4 Map

特点：以key、value的形式存放值，key不可重复

常用实现：`HashMap`、`LinkedHashMap`、`TreeMap`、`ConcurrentHashMap`

CloudMall 场景：通过id查询商品详情

```java
Map<Long, Product> productMap = new HashMap<>();
```

Key 和 Value 分别是：id和产品对象

## 4. HashSet 为什么能去重

`HashSet` 底层主要基于：HashMap实现的
添加元素时的过程：

1. 以元素作为key，统一占位符作为value,通过hashcode定位桶的下标
2. 如果桶内没有对象，则直接放进去
3. 如果桶内有对象，则说明哈希冲突，这时再通过equals()方法判断key是不是一样，不一样才放进去。

`hashCode()` 的作用：通常把业务的唯一值做哈希处理，通过哈希去定位桶的下标

`equals()` 的作用：比较key的值是否一样

为什么重写 `equals()` 时通常也要重写 `hashCode()`：
如果只重写equals()，当两个值一样但hashCode()不一样，就会出现同样的值存放了两次。

## 6. 算法题：最长连续序列

题目：给定一个未排序的整数数组，找出数字连续的最长序列长度。

示例：

```text
[100, 4, 200, 1, 3, 2] -> 4
```

### 6.1 TreeSet 解法

思路：通过TreeSet保证集合的排序，然后再便利TreeSet获取最长排序长度，但TreeSet在每次插入时都要维护树结构，时间复杂度不如HashSet。

时间复杂度：O(nlogn)
空间复杂度：O(n)

### 6.2 HashSet 优化

核心思路：只从连续序列的起点开始向后寻找。

判断起点的条件：

```java
!set.contains(num - 1)
```

为什么不能从每个数字都开始向后查找：
只有当这个数字没有前面的值，才能认为是排序的开始，从每个数字开始向后查找，会做不必要的判断。

时间复杂度：O(n)  
空间复杂度：O(n)

## 7. 面试题整理

答：Collection和Map，Collection里包含List、Set、Queue。

### 7.2 ArrayList 和 LinkedList 怎么选？

答：需要快速查询选择ArrayList，需要排序选择LinkedList

答：HashSet底层是由HashMap实现的，其保证元素不重复实际是由HashMap实现的，把元素作为key，再把统一占位符作为value。

答：如果只重写equals()，当两个值一样但hashCode()不一样，就会出现同样的值存放了两次。

### 7.5 HashMap 为什么不属于 Collection？

答：Collection是一组元素，Map是以键值形式存储数据的。

### 7.6 ArrayDeque 为什么常比 LinkedList 更适合作队列或栈？

答：虽然ArrayDeque和LinkedList都能实现队列和栈，但在维护上ArrayDeque只需要维护头尾下标，而LinkedList除了要维护当前节点还要维护前后节点，维护成本高。
