## 2. volatile 是什么

`volatile` 是 Java 的关键字，用于修饰共享变量。

核心结论：

```text
volatile：保证可见性和有序性，不保证原子性。
```

| 能解决 | 不能解决 |
| --- | --- |
| 一个线程修改后，其他线程看不到最新值 | `i++`、`stock--` 等复合操作的线程安全问题 |
| 危险的指令重排 | 多线程同时修改共享变量导致的超卖 |

## 3. 可见性：停止线程

普通变量：

```java
private static boolean running = true;
```

线程可能无法停止的原因：

```text
主线程把 running 改为 false；
工作线程可能仍然读取自己之前看到的 true；
工作线程无法及时看到最新值，因此循环不退出。
```

使用 `volatile`：

```java
private static volatile boolean running = true;
```

我的理解：

>主线程把running改为false后，工作线程及时看到了修改结果，然后退出循环

停止线程 Demo 路径：

```text
code/cloudmall/src/main/java/com/cloudmall/demo/thread/VolatileDemo.java
```

运行结果：

```text
修改 running 前：System.out.println(i++)
修改 running 后：System.out.println("线程已停止")
工作线程是否结束：是
```

注意：

> `volatile` 是协作式停止标记。若线程正在 `sleep()`、`wait()` 或 I/O 阻塞中，仅修改 `running` 不会让线程立即醒来，通常还需要使用 `interrupt()`。

## 4. 原子性：volatile 不能解决超卖

```java
private static volatile int stock = 10;
```

以下代码是否线程安全：

```java
stock--;
```

答案：

>不安全

原因：

```text
stock-- 包含：
1. 读取 stock
2. 计算 stock - 1
3. 写回 stock
```

多个线程可能同时读取相同的旧值，因此 `volatile` 虽然让线程能看到最新值，但不能保证“读、改、写”作为一个整体执行。

对比：

| 方案 | 可见性 | 原子性 | 能否解决库存扣减超卖 |
| --- | --- | --- | --- |
| 普通 `int` | 否 | 否 | 否 |
| `volatile int` | 是 | 否 | 否 |
| `synchronized` | 是 | 是 | 是 |
| `AtomicInteger` | 是 | 是 | 是 |

## 5. 有序性：指令重排

指令重排的含义：

>编译器、JIT 和 CPU 为了性能，可能调整代码执行顺序。只要单线程结果不变，这种优化就是允许的

`instance = new Singleton()` 可以粗略拆分为：

```text
1. 分配对象内存
2. 初始化对象
3. 将 instance 指向该对象
```

没有 `volatile` 时，可能出现：

```text
1 -> 3 -> 2
```

问题过程：

```text
线程 A：instance 已经不是 null，但对象尚未初始化完成。
线程 B：判断 instance != null，直接返回该对象。
结果：线程 B 可能拿到半初始化对象。
```

`volatile` 的作用：

>限制 volatile 变量读写前后可能破坏并发正确性的指令重排，防止对象先被赋值、后初始化。

## 6. 双重检查锁单例

```java
public class Singleton {

    private static volatile Singleton instance;

    private Singleton() {
    }

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

| 代码部分 | 作用                                             |
| --- |------------------------------------------------|
| 第一层 `if` | 防止每次获取实例都要竞争锁                                  |
| `synchronized` | 锁住 Singleton.class 对象，保证同一时刻只有一个线程进入创建实例的同步代码块 |
| 第二层 `if` | 防止多线程通过第一层if后重复创建实例                            |
| `volatile instance` | 保证实例创建后的可见性，并防止 instance 引用先发布、对象后初始化的危险重排     |

我的总结：

>双重检查锁通过第一层 if 避免每次获取实例都加锁，通过第二层 if 防止多个线程依次进入同步块后重复创建对象。synchronized 保证创建过程互斥，volatile 保证实例对其他线程可见，并禁止 instance = new Singleton() 发生导致半初始化对象暴露的危险重排

## 8. 高频面试题

### 8.1 volatile 有什么作用？

>可以保证可见性和有序性，可见性是指一个线程完成操作后其结果能被其他线程及时看到，有序性是指禁止关键代码发生指令重排，防止对象先被赋值，后被初始化

### 8.2 volatile 能保证原子性吗？为什么？

>不能，不能让 stock--、i++ 这类读取、修改、写回组成的复合操作变成原子操作，多个线程仍可能交错执行

### 8.3 普通 boolean 为什么可能无法停止线程？

>当一个线程把boolean变量修改之后，其他线程可能无法及时看到其修改后的结果，无法跳出循环，导致无法停止线程

### 8.4 volatile 为什么能禁止指令重排？

>volatile会限制关键操作在多线程之间被错误地重排，从而避免其他线程看到“不完整的数据”

### 8.5 双重检查锁为什么必须加 volatile？

>保证实例创建后对其他线程可见，并防止 instance 引用先赋值、对象后初始化，导致其他线程拿到半初始化对象

### 8.6 volatile 和 synchronized 的区别？

>volatile只能保证可见性和有序性，不能保证原子性，synchronized可以保证可见性、原子性和有序性
