## 1. 总目标与完成标准

本计划不以“学完多少知识点”为目标，而是以能写、能讲、能排查、能落地为完成标准。

### 目标

- 在 8 周内补齐 Java 后端中级岗位的高频面试能力。
- 将 CloudMall 从基础骨架逐步扩展为商品、库存、订单、缓存、消息的可讲解项目。
- 从第 3 周开始同步投递和模拟面试，根据面试反馈动态调整优先级。
- 不虚构生产经历；涉及线上调优、分布式改造等内容，以个人项目、压测或故障演练的方式如实表达。

### 验收标准

1. 每个核心主题有至少一个可运行 Demo 或 CloudMall 功能。
2. 每个主题能用“结论 -> 原理 -> 场景 -> 取舍”在 2-3 分钟内说明。
3. 能说明 JVM、Linux、MySQL、Redis 常见问题的基础排查路径。
4. CloudMall 至少形成商品、库存、订单、缓存、MQ 的完整业务链路。
5. 每周完成 3-5 道高频算法题，并写清复杂度与优化思路。

## 5. CloudMall 实施路线

### 阶段一：并发库存与异步任务

- 用多线程模拟库存超卖。
- 对比 synchronized、AtomicInteger、后续 Lock 的适用场景。
- 使用线程池执行商品批量任务或异步任务。
- 记录单 JVM 方案的边界：多实例下需要数据库条件更新、分布式锁等方案。

### 阶段二：商品与订单基础链路

- 商品 CRUD：Controller、Service、Mapper、MySQL。
- 订单创建与库存扣减。
- 商品、库存、订单表设计与索引设计。
- 使用事务保证本地数据一致性。

### 阶段三：缓存与消息链路

- 商品详情采用 Cache Aside。
- 演练缓存穿透、击穿、雪崩的应对方案。
- 下单后投递订单创建消息。
- 消费端实现幂等、重试和死信处理。

### 阶段四：服务化与交付

- 网关路由、Feign 调用与基础服务拆分。
- Docker 化部署。
- 准备项目架构图、调用链路图和项目面试表达。

## 6. 算法安排

每周 3-5 题，优先覆盖面试高频题型：

| 题型 | 学习重点 |
| --- | --- |
| Hash | 去重、计数、映射、连续序列 |
| 双指针 | 原地数组操作、两端收缩 |
| 链表与队列 | 反转、合并、快慢指针、栈队列 |
| 二分 | 边界、循环不变量、答案区间 |
| 滑动窗口 | 子串、连续子数组、窗口收缩 |
| 栈与单调栈 | 括号匹配、最近更大/更小元素 |

算法笔记必须记录：题意、朴素思路、优化思路、时间复杂度、空间复杂度、适用模板。

## 7. 面试与复盘机制

### 从第 3 周开始

- 每周至少投递并参加 1 次模拟面试或真实面试。
- 将不会的问题记录为“面试反馈”，在 48 小时内补齐笔记、Demo 或项目场景。

1. 本周可以独立讲清的问题。
2. 本周仍然模糊的问题。
3. CloudMall 新增了什么能力。
4. 下周最关键的一个补短板目标。
5. 算法题型是否完成了归纳。
