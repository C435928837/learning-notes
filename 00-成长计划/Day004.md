## 1. 单例模式为什么出现
1. 单例模式解决什么问题？
保证整个JVM只存在一个实例，避免重复创建对象造成资源浪费以及对象状态不统一的问题。
2. 哪些对象适合做成单例？
全局配置类、线程池
3. 哪些对象不适合做成单例？
用户对象、订单对象

##2. 饿汉式
1. 饿汉式什么时候创建对象？
类加载的时间创建
2. 为什么线程安全？
在类加载的时候就完成创建实例的动作，后续获取实例时直接返回实例。
3. 优点是什么？
线程安全
4. 缺点是什么？
对于创建实例的成本高的对象，在不必要创建时会造成资源浪费。

##3. 懒汉式
1. 懒汉式什么时候创建对象？
调用类的获取实例方法时。
2. 为什么普通懒汉式线程不安全？
多个线程同时获取实例时，可能会创建多个实例。
3. 它的优点和缺点是什么？
优点是需要才创建，节省内存空间，缺点是线程不安全。

##4. 双重检查锁
1. 为什么需要 synchronized？
防止创建多个实例
2. 为什么需要两层 if？
第一层是避免每次获取实例都要等待锁释放；第二层是判断实例是否已经被其他线程创建。
3. 为什么 instance 要加 volatile？
volatile 禁止指令重排，避免其他线程看到 instance != null 时，拿到一个还没有初始化完成的对象。

##静态内部类
1. 静态内部类什么时候创建对象？
调用类的获取实例方法时。
2. 为什么能懒加载？
外部类加载时，静态内部类不会立即加载。只有调用 getInstance() 使用到内部类时，JVM 才加载内部类并创建实例。
3. 为什么线程安全？
内部类由JVM保证线程安全
