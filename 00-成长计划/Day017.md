## 2. HashMap、Hashtable、ConcurrentHashMap 对比

| 对比项 | `HashMap` | `Hashtable`  | `ConcurrentHashMap` |
| --- |-----------|--------------|---------------------|
| 线程安全 | 否         | 是            |                     |
| 并发方式 | 无         | synchronized | 锁痛                  |
| 并发性能 | 高         | 低            | 高                   |
| 是否允许 null | 允许一个null  | 不允许          | 不允许                 |
| 使用建议 | 单线程       | 不推荐使用        | 多线程                 |

我的结论：

>不需要考虑线程安全的选择HashMap，需要考虑线程安全的选择ConcurrentHashMap，不推荐使用HashTable

## 3. 为什么 get + put 不安全

不安全写法：

```java
map.put(productId, map.get(productId) + 1);
```

并发过程：

```text
初始浏览量：0

线程 A：获取浏览量为0
线程 B：获取浏览量为0
线程 A：浏览量加1
线程 B：浏览量加1

最终结果：1
正确结果：2
```

问题原因：

>它们是两次独立调用，组合起来不具备原子性；即使用的是 ConcurrentHashMap，也会丢失更新

## 4. ConcurrentHashMap 的 JDK 8 实现

### 4.1 空桶插入：CAS

CAS 的含义：

>比较并赋值，只有当前值仍等于预期值，才把新值赋给当前值

空桶插入流程：

```text
桶中当前值：null
预期值：null
新值：1
CAS 成功后：赋值1
CAS 失败后：不赋值
```

### 4.2 桶非空：锁桶头

```java
synchronized (f) {
    // 操作当前桶
}
```

`f` 是：

>桶内的头节点

为什么不是锁整张 Map：

>并发效率低，多线程操作不一定都是同一个key

不同 Key 是否一定可以并行操作：

>不同 Key 可能哈希冲突到同一个桶，仍会竞争该桶的锁，不同桶通常可以并行操作

### 4.3 读操作

`ConcurrentHashMap.get()` 通常不加锁的原因：

>因为有volatile修饰，变量可以看到其他线程操作后的值

`volatile` 在这里保证：

>可见性

遍历 `ConcurrentHashMap` 时是否是完整一致的快照：

>不是

## 5. 常用原子 API

### 5.1 putIfAbsent()

```java
map.putIfAbsent(key, value);
```

作用：

>如果key不存在就放value进入，key已存在则返回旧值

适用场景：

>初始化一次性变量

### 5.2 computeIfAbsent()

```java
map.computeIfAbsent(key, ignored -> new LongAdder());
```

作用：

>如果key不存在就生成LongAdder对象并放进入，key已存在则返回对象

CloudMall 场景：

>按需生成制定对象

### 5.3 merge()

```java
map.merge(productId, 1, Integer::sum);
```

作用：

>Key 不存在时放入给定值；已存在时，使用旧值和新值执行合并函数并写回结果

适用场景：

>多线程运算

## 7. 算法题

题目：字母异位词分组

题型：

>给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

核心思路：

>统计每个字母出现的次数，并按字母顺序加次数排列作为key，key相同的就是异位词

Key 的设计：

>字母加次数

为什么 Key 的生成顺序必须固定：

>保证异位词的key相同

时间复杂度：O(n * k)，n 是字符串数量，k 是平均字符串长度  
空间复杂度：O(n * k)，n 是字符串数量，k 是平均字符串长度

## 8. 高频面试题

### 8.1 HashMap 为什么不能用于并发写？

>HashMap的操作没有加锁，线程不安全

### 8.2 Hashtable 和 ConcurrentHashMap 的区别？

>Hashtable通过synchornied实现线程安全，ConcurrentHashMap通过锁桶实现线程安全

### 8.3 JDK 8 ConcurrentHashMap 如何实现高并发？

>空桶时插入用CAS，相同的key通过对桶内的头节点加锁实现线程安全，不同的key允许同时操作，get()不加锁

### 8.4 为什么 ConcurrentHashMap 不允许 null Key 和 null Value？

>因为在get()返回null时会存在歧义，无法判断正确情况

### 8.5 putIfAbsent、computeIfAbsent、merge 如何选择？

>putIfAbsent可用于初始化一次性变量，computeIfAbsent可用于按需生成制定对象，merge适合多线程运算
